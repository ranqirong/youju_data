# ERP 技能修复报告 - 字段名空格问题

## 📅 修复日期
2026-03-27

## 🐛 问题描述

### 根本原因
1. **字段名不匹配**：ERP 导出的原始 Excel 文件中，列名**没有空格**（如 `"项目部层级 1"`），但脚本中硬编码的字段名**有空格**（如 `"项目部层级 1"`），导致字段名不匹配，数据读取失败。

2. **数据读取逻辑缺陷**：原始代码只读取 `v` 元素的值，但部分单元格使用 `is/t`（inline string）格式存储，导致这些数据被忽略。

### 影响范围
- 所有包含"项目部层级"、"渠道部层级"、"授权渠道部层级"字段的数据处理
- 每日数据报表和每周数据报表生成
- 数据统计和聚合结果不准确

## ✅ 修复内容


### 2. 数据读取逻辑修复

**修复前**（只读取 v 元素）：
```python
values = [cell.find('ss:v', ns).text if cell.find('ss:v', ns) is not None else '' 
          for cell in cells]
```

**修复后**（支持 v 和 is/t 两种格式）：
```python
values = []
for cell in cells:
    val = ''
    v_elem = cell.find('ss:v', ns)
    if v_elem is not None and v_elem.text:
        val = v_elem.text
    else:
        t_elem = cell.find('ss:is/ss:t', ns)
        if t_elem is not None and t_elem.text:
            val = t_elem.text
    values.append(val)
```
