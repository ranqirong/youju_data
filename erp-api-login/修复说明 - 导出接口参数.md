## 🐛 问题描述

新房合同导出接口的请求参数中，`date`、`signTmStart`、`signTmEnd` 三个时间参数没有正确区分：

**修复前**:
```json
{
  "date": [start_time, end_time],
  "signTmStart": start_time,  // ❌ 与 date[0] 相同
  "signTmEnd": end_time       // ❌ 与 date[1] 相同
}
```

**根据抓包数据的正确格式**:
```json
{
  "date": [1774195200000, 1774540800000],  // 查询日期范围
  "signTmStart": 1774195200000,            // 签约开始时间
  "signTmEnd": 1774627199999               // 签约结束时间
}
```

### 时间戳计算

```python
from datetime import datetime

# 日期：2026-03-26
start_dt = datetime(2026, 3, 26, 0, 0, 0)
end_dt = datetime(2026, 3, 26, 23, 59, 59)

# 转换为毫秒时间戳
query_start_ts = int(start_dt.timestamp() * 1000)  # 1774454400000
query_end_ts = int(end_dt.timestamp() * 1000)      # 1774540799000

# 签约时间范围（通常与查询日期范围相同）
sign_tm_start = query_start_ts
sign_tm_end = query_end_ts
```

## 📝 注意事项

1. **时间戳精度**: 所有时间戳使用**毫秒**精度（乘以 1000）
2. **结束时间**: 日期范围的结束时间应为当天的 `23:59:59`
3. **参数独立性**: `signTmStart` 和 `signTmEnd` 现在可以独立设置，支持与查询日期范围不同
4. **向后兼容**: 如果不传 `sign_tm_start` 和 `sign_tm_end`，默认与查询日期范围相同
